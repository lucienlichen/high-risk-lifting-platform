
一、项目定位
这是一个工业互联网平台前端演示原型，面向高风险起重装备的智能运维与风险防控业务。当前版本按照“领导建议版”的页面结构来实现，重点突出：
- 顶部标题栏
- 左侧场景导航
- 右侧主内容区
- 主内容区分为数据分类区、核心内容区、服务菜单区
- 支持从场景进入设备列表，再进入设备详情页
- 页面结构清晰，适合项目汇报和演示

二、整体信息结构
平台整体采用以下结构：

1. 顶部标题栏
- 显示平台名称：“高风险起重装备智能运维与风险防控服务平台”
- 可显示平台副标题、时间、搜索框、消息通知、用户头像、用户名、退出按钮
- 顶部栏是全局固定区域

2. 左侧场景导航
固定显示四大场景：
- 冶金
- 港口
- 建筑
- 造船

点击左侧不同场景时，右侧内容切换到对应场景下的设备总览页面。

3. 右侧内容区域
右侧内容区域用于展示场景设备总览和设备详情相关内容。

三、页面与交互逻辑
需要实现以下 4 个核心页面 / 状态：

1. 首页-设备卡片视图版
2. 首页-设备列表视图版
3. 场景设备总览页
4. 设备数据与服务主页面（设备详情页）

四、关键结构要求
1. 顶部标题栏
- 全局固定
- 位于页面最上方
- 深蓝色横向标题栏
- 显示平台名称、时间、搜索、消息、用户信息

2. 左侧导航
- 位于页面左侧
- 使用大图标 + 场景名称的纵向导航形式
- 当前选中场景高亮
- 风格应明显区别于普通窄侧边栏，更像“场景切换面板”

3. 设备详情页布局要求（非常重要）
设备详情页必须采用以下顺序：

- 第一层：顶部标题栏（全局）
- 第二层：数据分类菜单区（位于内容区域最上方，也就是紧贴在顶部标题栏下方）
- 第三层：核心内容展示区
- 第四层：底部服务菜单区

注意：
- 数据分类菜单不能放在页面中间，也不能放在核心内容区下方
- 数据分类必须是内容区最上方的一级切换区域
- 底部服务菜单必须始终显示，即使当前没有选择具体设备，也要显示服务菜单，只是内容区可以提示“请选择设备后查看详细分析内容”

4. 服务菜单显示规则（非常重要）
- 服务菜单始终显示在右侧主内容区的最下方
- 不选择设备时：
  - 服务菜单仍然可见
  - 可为未激活状态或半激活状态
  - 核心内容区显示提示信息，例如“请先选择设备，再查看智能运维与风险防控服务详情”
- 选择设备后：
  - 服务菜单正常可点击
  - 与当前设备详情联动

五、页面详细要求

页面 1：首页-设备卡片视图版
页面功能：
- 作为平台入口页
- 展示当前选中场景下的设备接入情况
- 使用卡片形式展示设备

页面结构：
1. 顶部标题栏
2. 左侧场景导航
3. 右侧内容区
   - 当前场景名称与简介
   - 场景统计卡片：
     - 设备总数
     - 在线设备数
     - 高风险设备数
     - 当日告警数
   - 视图切换按钮：
     - 卡片视图
     - 列表视图
   - 设备卡片墙
   - 底部服务菜单区（始终显示）

设备卡片字段：
- 设备名称
- 设备编号
- 当前状态
- 风险等级
- 最近告警时间

交互：
- 点击设备卡片可进入设备详情页
- 当前页面中卡片视图为选中态
- 不显示表格列表

页面 2：首页-设备列表视图版
页面功能：
- 与设备卡片视图版结构一致
- 仅设备展示形式改为表格列表

页面结构：
1. 顶部标题栏
2. 左侧场景导航
3. 右侧内容区
   - 当前场景名称与简介
   - 场景统计卡片
   - 视图切换按钮
   - 设备表格列表
   - 底部服务菜单区（始终显示）

表格字段：
- 设备名称
- 设备编号
- 设备类型
- 当前状态
- 风险等级
- 最近告警时间
- 操作按钮

交互：
- 点击列表行或操作按钮进入设备详情页
- 当前页面中列表视图为选中态
- 不显示设备卡片墙

页面 3：场景设备总览页
页面功能：
- 当点击左侧场景后，展示该场景下接入平台的设备总览
- 支持设备筛选、状态查看和进入设备详情

页面结构：
1. 顶部标题栏
2. 左侧场景导航
3. 右侧内容区
   - 当前场景名称
   - 场景简介
   - 统计卡片：
     - 设备总数
     - 在线设备数
     - 离线设备数
     - 高风险设备数
     - 当日告警数
   - 筛选区：
     - 设备名称
     - 风险等级
     - 设备状态
     - 设备类型
   - 设备列表区域（表格为主）
   - 辅助信息区域：
     - 风险设备排行
     - 最近告警设备
     - 场景设备状态分布图
   - 底部服务菜单区（始终显示）

交互：
- 点击设备进入设备详情页
- 不选择设备时，底部服务菜单仍显示

页面 4：设备数据与服务主页面（设备详情页）
页面功能：
- 展示单台设备的多类数据和智能服务
- 这是平台最核心的页面

页面结构要求（严格执行）：
1. 顶部标题栏（全局固定）
2. 数据分类菜单区（位于内容区域最上方，紧贴顶部标题栏下方）
3. 核心内容展示区
4. 服务菜单区（位于内容区域最下方，并始终显示）

详情页还需要保留左侧场景导航。

数据分类菜单包括以下 10 项：
- 点巡检数据
- 故障数据
- 维修数据
- 检测数据
- 检验数据
- 运行数据
- 监控数据
- 监测数据
- 基础信息
- 出厂数据

要求：
- 数据分类菜单采用横向标签/Tab 形式
- 位于右侧内容区最顶部
- 默认选中“运行数据”或“基础信息”
- 切换不同分类时，中间核心内容区跟随变化

核心内容区建议：
- 顶部指标卡
- 中部趋势图表
- 下部数据表格
- 可选的右侧设备摘要卡片

默认以“运行数据”为例展示：
- 运行时长
- 当前负载
- 健康评分
- 告警次数
- 运行趋势图
- 监测曲线图
- 历史记录表格

服务菜单包括以下 7 项：
- 安全监控
- 健康评估
- 裂纹预测
- 故障诊断
- 风险监测
- 预警处置
- 决策建议

服务菜单要求：
- 固定显示在内容区底部
- 不选择设备时也显示
- 选择设备后可点击并展示分析内容
- 当前未选设备时，可显示提示状态
- 视觉上要和数据分类菜单区分开

七、统一设计风格（必须直接落实到实现中）
请采用以下统一设计风格进行实现：

整体风格定位：
浅色科技风企业平台 + 轻毛玻璃设计 + 深蓝色顶部标题栏 + 左侧大图标场景导航

整体风格要求：
- 页面整体呈现高端企业级工业数字平台气质
- 主体内容区以浅色为主，界面简洁、通透、理性、专业
- 保持清晰的信息层级、大面积留白、规整的卡片化布局
- 视觉上有轻科技感和轻未来感，但必须克制，不花哨、不炫技
- 不要做成深色大屏风，也不要做成概念化科技海报
- 要更像真实可落地的工业互联网平台和企业级后台系统

顶部标题栏风格要求：
- 页面顶部采用深蓝色横向标题栏，形成稳定、专业、可信的视觉锚点
- 顶部栏应具有企业级平台全局导航的感觉，简洁规整，识别度强
- 标题栏内文字建议使用白色或浅色，保证对比清晰
- 可容纳平台名称、副标题、时间、搜索框、消息通知、用户头像等信息
- 顶部栏不要使用复杂发光、厚重阴影和过强装饰，整体偏克制、现代、专业

左侧导航风格要求：
- 左侧采用纵向场景导航布局
- 导航项由大尺寸线性图标 + 场景名称组成，具有明显的行业场景识别性
- 图标风格简洁统一，适合工业场景，例如起重机、港口设备、建筑吊装、造船设备等
- 导航整体更像“场景专题切换面板”，而不是传统细窄后台菜单
- 当前选中项有明确高亮状态，但仍保持简洁、克制和专业
- 左侧导航背景可使用浅灰白色或轻微毛玻璃面板，增强层次感

主体内容区风格要求：
- 主体内容区采用浅色背景，以白色、浅灰蓝、浅蓝灰为主
- 使用大量留白和规整的模块划分，提升阅读舒适性和高级感
- 内容容器采用浅色卡片设计，带轻阴影、轻描边和适度圆角
- 可以加入轻微毛玻璃质感，但必须非常克制，只用于卡片、面板、筛选区、图表容器等
- 卡片排版简洁有序，强调数据清晰、结构规整和模块化表达
- 图表区域、统计卡片、设备卡片、列表容器需保持统一的设计语言
- 内容区整体应体现简洁、理性、可信的企业平台感

毛玻璃与细节要求：
- 使用轻度 glassmorphism，而不是强烈玻璃拟态
- 面板可以带少量背景模糊、半透明感和柔和阴影
- 边框采用低对比度浅色描边
- 圆角中等偏大，但不要夸张
- 避免大面积渐变、复杂高光、霓虹边框和炫目发光效果
- 所有细节都应服务于“专业、克制、精致”的目标

色彩要求：
- 主背景色：白色、浅灰蓝、浅蓝灰
- 顶部栏主色：深蓝色
- 主高亮色：蓝色、青蓝色
- 正常状态：绿色
- 预警状态：橙色
- 高风险状态：红色
- 整体颜色控制简洁，不要高饱和堆砌，不要颜色过多

排版与气质要求：
- 排版清晰，留白充足，层级明确
- 字体现代、干净、易读
- 图标采用简洁线性科技风
- 页面整体气质偏“高端工业智能平台、企业级数字中台、专业运营管理系统”
- 界面应让人感觉稳定、可信、清爽、现代，而不是炫酷或浮夸
